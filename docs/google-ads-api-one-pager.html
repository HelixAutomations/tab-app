1) Overview

We will use the Google Ads API strictly for read‑only reporting to power an internal dashboard that calculates marketing ROI alongside Google Analytics 4 and internal business data. No ad creation, edits, bidding, or budget changes will be performed via the API.

2) Scope and capabilities

  • Capability: Reporting only (read‑only).
  • Campaign types queried: Search only (Video not used).
  • Data: daily metrics (impressions, clicks, cost_micros, conversions, ctr, cpc, cpa).
  • Audience: internal staff only; not customer‑facing; no data resale.

3) Architecture and authentication

  • Backend service queries the Google Ads API (v22) using GAQL Search over HTTPS.
  • OAuth 2.0 installed‑app/web flow with a user belonging to the MCC; refresh token stored securely.
  • Developer token issued to MCC (login‑customer‑id = 7748108809) used for reporting.
  • Data is aggregated by day and combined with GA4 for internal analytics.

4) Data protection and security

  • Secrets (developer token, client id/secret, refresh token) stored in Azure Key Vault; never in source control.
  • Least‑privilege access; tokens encrypted at rest; TLS 1.2+ in transit.
  • No collection of personally identifiable information (PII) from Google Ads responses.
  • Operational logs exclude tokens and any sensitive values; periodic token rotation.

5) Compliance and limits

  • Usage adheres to Google Ads API Terms of Service and policies.
  • Low‑volume reporting (dashboard refresh), well within Basic access quotas.
  • No write actions; no user data export or resale.
